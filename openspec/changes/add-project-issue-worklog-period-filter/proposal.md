## Why

В проектном списке задач Plane есть штатный rich filter, но в нем нельзя отобрать задачи по периоду внесения фактических часов. Из-за этого пользователь не может ответить на базовый вопрос "какие задачи получили факт в выбранном периоде" и вынужден считать это вне продукта.

Сейчас колонка `Факт` всегда показывает суммарные часы по задаче за все время, что делает периодный анализ неточным. Для отчетных и операционных сценариев нужен единый фильтр, который отбирает задачи по записям факта в диапазоне и показывает значение `Факта` именно за этот диапазон.

## What Changes

- Добавить в штатный rich filter проектной страницы задач новый date-range фильтр по дате записи факта часов.
- Фильтровать задачи по существованию хотя бы одной неудаленной записи worklog в выбранном диапазоне.
- При активном фильтре пересчитывать `actual_hours` для issue payload только по worklog, попавшим в диапазон.
- Исключать из результата задачи, у которых нет worklog в выбранном диапазоне.
- Сохранить текущее поведение для всех остальных фильтров и для случаев, когда новый фильтр не задан.

## Capabilities

### New Capabilities
- `project-issue-worklog-period-filter`: фильтрация задач проекта по диапазону дат записей факта с периодным пересчетом значения `Факт`.

### Modified Capabilities
- None.

## Impact

- Backend issue query pipeline в `apps/api/plane/app/views/issue/base.py`.
- Rich filter allowlist и filterset в `apps/api/plane/utils/filters/filterset.py`.
- Типы и frontend-конфигурация rich filters в `packages/types` и `apps/web/ce/hooks/work-item-filters`.
- Конфигурация доступных фильтров для project issues.
- Модель и индексация `ext_worklogs`.
